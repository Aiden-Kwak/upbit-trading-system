# Upbit Trading System — Claude Instructions

## 프로젝트 목적

업비트 KRW 마켓 대상 자동매매 시스템. Toss Trading System과 **완전히 독립** (별도 DB, 별도 설정, 별도 프로세스).

## 수정 전 필수 참조

- `docs/strategy.md` — 3종 전략 (VB/MB/MR) 상세 + 파라미터 근거
- `docs/architecture.md` — 사이클 흐름, 주문 실행, DB 스키마
- 이 문서를 **읽지 않고 코드 수정 금지**

## 핵심 원칙

1. **드라이런 우선**: 기본 매수 주문은 반드시 `dry_run=True` 확인 후 실행
2. **API 레이트리밋**: 주문 초당 8회 / 분당 200회. 시세 초당 10회 / 분당 600회. 레이트리밋 위반 시 Upbit 계정 페널티
3. **출금 권한 금지**: API 키는 "자산 조회 + 주문 조회/생성/취소" 까지만
4. **24/7 인식**: 시장 열림/닫힘 개념 없음. 새벽 유동성 얕은 시간대 신규매수 자제
5. **KRW 기준**: 모든 가격·평가·PnL은 원화 단위. 환율 변환 없음
6. **pyupbit 의존**: 직접 REST 호출 대신 pyupbit 래퍼 사용. 단, 레이트리밋·재시도는 `upbit_client.py`에서 관리
7. **BTC 레짐 우선**: 개별 코인 시그널보다 BTC 상태 게이트 먼저. BTC 급락 시 모든 매수 차단

- `data/trading.db` 가 primary 저장소
- `trades`, `daemon_cycles`, `signals`, `errors`, `equity_curve` 테이블
- 모든 거래는 DB 즉시 반영 (파일 백업은 백그라운드)

## 전략 게이트 순서 (절대 순서)

1. 세션/API 키 검증
2. **BTC 레짐 체크** — 여기서 차단되면 나머지 스킵
3. 일일 손실 한도
4. 주간 MDD
